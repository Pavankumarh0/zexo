# Zexo — Mobile (Flutter)

Proximity-first social discovery client. **Flutter 3.x + Riverpod + go_router**, talking to
the [Zexo API](../backend) and Supabase.

Implements the screens and flows defined in [`.kiro/specs/zexo`](../.kiro/specs/zexo) and
follows the steering rules in [`.kiro/steering`](../.kiro/steering).

## Layout (feature-first)

```
lib/
  main.dart                       # bootstrap: Supabase init, Sentry, router
  core/
    config/app_config.dart        # --dart-define config + AppConstants
    theme/app_theme.dart          # Material 3 theme + shared-interest highlight
    api/                          # Dio client + error envelope
    ws/ws_client.dart             # chat-thread WebSocket
    location/                     # geolocator service + 30s polling controller
    router/                       # go_router shell + redirects
    telemetry/                    # Sentry init
    providers.dart                # config / Supabase / auth / api providers
  shared/
    models/                       # immutable models with fromJson
    widgets/                      # DistanceBadge, TagChip, EmptyState
  features/
    onboarding/                   # phone OTP + profile setup + location permission
    discover/                     # ranked cards feed + map placeholder
    chat/                         # thread list + real-time thread
    events/                       # nearby events + inline RSVP
    profile/                      # public profile, visibility, profile repo
    settings/                     # radius, visibility, account, privacy
```

## Run

This app reads configuration from `--dart-define` values (never hardcoded secrets):

```bash
cd mobile
flutter pub get
flutter run \
  --dart-define=API_BASE_URL=https://your-api.up.railway.app \
  --dart-define=WS_BASE_URL=wss://your-api.up.railway.app \
  --dart-define=SUPABASE_URL=https://your-project.supabase.co \
  --dart-define=SUPABASE_ANON_KEY=your-anon-key \
  --dart-define=MAPBOX_TOKEN=your-mapbox-token \
  --dart-define=SENTRY_DSN=your-sentry-dsn
```

invisible mode, nearby events with RSVP, chat list, real-time chat thread, settings, public
profile, privacy screen, and the navigable bottom-nav shell.
